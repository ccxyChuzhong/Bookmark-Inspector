# 📈 Bookmark Inspector 升级指南

## 🎉 v3.0.0 重大更新

### 🆕 新增功能

#### 1. 书签列表状态徽标完善
- 每条书签都会显示明确状态：`可用 / 失效 / 检测中 / 未检测`
- 从未检测过的书签会显示中性的 **"未检测"** 徽标，避免误以为正在检测
- 文件夹项不再显示状态徽标，仅书签项展示

#### 2. 书签行内删除
- 列表中每一条书签的行尾追加**图标删除按钮**（垃圾桶）
- 默认隐藏，鼠标悬停在该行时才浮现，不占用视觉空间
- 点击二次确认后直接删除该书签，并实时更新计数

#### 3. WebDAV 同步
- 通过标准 WebDAV 协议把书签备份到自有网盘
- 支持「测试连接 / 查询远端 / 备份 / 恢复」四个操作
- 设置页面集中展示远端备份时间、最近拉取与最近备份时间

#### 4. 树形文件夹下拉
- 文件夹下拉框升级为**可折叠的树形结构**
- 多层级文件夹下展开/收起更直观
- 兼顾紧凑展示与查找效率

#### 5. 紧凑配置页布局
- 重新组织 `settings.html` 的留白、字号与控件尺寸
- 代理、并发、WebDAV 三大模块在常规屏幕上**一屏可见**

### 🐛 关键缺陷修复

#### 1. 检测进度条不显示
- **现象**：v2.x 中点击"检测网址可用性"后，进度条不会出现
- **原因**：`updateSectionVisibility()` 条件为 `hasResults && !isChecking`，检测过程中反而隐藏了结果区
- **修复**：改为 `isChecking || hasResults`——检测中或已有结果均显示

#### 2. 残留"检测中"徽标
- **现象**：未发起新检测的书签项也显示"检测中"
- **原因**：`startCheck` 会预先把所有书签写入 `bookmarkResultsById = { status: null }`，被取消/中断后这些 null 状态会持久化
- **修复**：仅当真正处于检测中、或已存在真实结果时才显示对应徽标，否则显示"未检测"

### 🎨 UI 设计焕新（继 v2.1）
- 全面切换到 **暖色调"Warm Linen"主题**
- 主面板宽度收窄，密度提升，更适合 Popup 窗口
- 阴影、圆角、过渡时间统一通过 CSS 变量管理

### 🔧 技术改进
- `js/popup.js` 中的两份重复 `renderBookmarksList` 同步更新，保证旧代码路径也享受新行为
- 新增 `js/webdav.js`，集中处理 WebDAV 同步逻辑
- 紧凑布局通过统一缩减 `padding / margin / 字号 / 控件高度` 实现，便于以后再次微调

### 📋 升级步骤

#### 自动升级（推荐）
通过浏览器商店安装的用户会自动更新到最新版本。

#### 手动升级
1. 备份当前书签（可选；本扩展自带导出功能）
2. 下载/拉取最新版本源码
3. 在浏览器扩展管理页面移除旧版本，加载新版本目录

### 🔄 兼容性说明

#### 向后兼容
- **书签数据**：完全兼容旧版本
- **导入/导出文件**：旧版本导出的文件可继续使用
- **配置持久化**：旧的 `chrome.storage.local` 数据可继续读取

#### 注意事项
- 旧版本中残留的 `status: null` 结果记录在 v3.0 不会再被错误展示为"检测中"。如果你需要清理，可直接重新发起一次检测覆盖。
- **权限要求**：无新增权限。

### 🎯 使用新功能

#### 行内删除书签
1. 在主面板选择一个文件夹，下方列出该文件夹的书签
2. 把鼠标悬停在任意一条书签上，行尾会浮现垃圾桶图标
3. 点击图标，确认对话框后该书签被移除

#### WebDAV 备份/恢复
1. 打开设置页（齿轮图标）
2. 在「WebDAV 同步」卡片填写服务器地址、账号、密码与远程路径
3. 点击「测试连接」验证可达性，然后用「备份」或「恢复」操作

#### 状态徽标速读
| 徽标 | 含义 |
| --- | --- |
| 可用 | 上次检测可访问 |
| 失效 | 上次检测不可访问 |
| 检测中 | 当前正在被检测 |
| 未检测 | 从未检测过该书签 |
